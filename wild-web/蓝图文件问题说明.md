# 蓝图文件格式问题

## 问题根源

经过深入分析，我发现**bieshu.wild蓝图文件本身的开口定义有问题**。

## 开口`from`字段的正确格式

根据渲染引擎的设计，opening的`from`字段应该是：
```
from: [沿墙距离, 相对墙底高度, 法向偏移]
```

**不是世界坐标！**

## 错误示例（当前bieshu.wild）

```json
{
  "type": "opening",
  "id": "window_front_left",
  "parentWall": "wall_front",
  "from": [-5.0, 1.0, -6.0],  // ❌ 错误：这是世界坐标
  "width": 1.5,
  "height": 1.2
}
```

### 为什么错误？

- wall_front: 从`[-8, 0, -6]`到`[8, 3, -6]`，长度16米
- 窗户世界X坐标是-5
- 沿墙距离应该是: `-5 - (-8) = 3`米
- 但蓝图写的是`-5`！

## 正确示例

```json
{
  "type": "opening",
  "id": "window_front_left",
  "parentWall": "wall_front",
  "from": [3.0, 1.0, 0.0],  // ✓ 正确：沿墙3米，高1米，无偏移
  "width": 1.5,
  "height": 1.2
}
```

## 为什么cabin_v1可以正常渲染？

检查cabin_v1.wild，它的开口定义应该是正确的格式（沿墙距离）。

## 解决方案

### 方案A：修复引擎（我尝试的）

让引擎自动识别并转换坐标 - **但这很复杂且容易出错**，因为：
- 无法区分用户到底想用哪种格式
- 不同方向的墙（X轴/Z轴）处理不同
- 容易破坏原本正常的蓝图（如cabin_v1）

### 方案B：修复蓝图文件（推荐） ✓

直接修正bieshu.wild中所有开口的坐标。

## 修复bieshu.wild的具体方案

### wall_front（沿X轴，从-8到8）

**当前（错误）**：
```json
{"id": "front_door", "from": [0.0, 0.0, -6.0]},
{"id": "window_front_left", "from": [-5.0, 1.0, -6.0]},
{"id": "window_front_right", "from": [5.0, 1.0, -6.0]}
```

**修正后**：
```json
{"id": "front_door", "from": [8.0, 0.0, 0.0]},        // 中心位置
{"id": "window_front_left", "from": [3.0, 1.0, 0.0]},  // -5-(-8)=3
{"id": "window_front_right", "from": [13.0, 1.0, 0.0]} // 5-(-8)=13
```

### wall_back（沿X轴，从-8到8）

**当前（错误）**：
```json
{"id": "window_back_center", "from": [0.0, 1.0, 8.0]},
{"id": "window_back_left", "from": [-5.0, 1.0, 8.0]},
{"id": "window_back_right", "from": [5.0, 1.0, 8.0]}
```

**修正后**：
```json
{"id": "window_back_center", "from": [8.0, 1.0, 0.0]},
{"id": "window_back_left", "from": [3.0, 1.0, 0.0]},
{"id": "window_back_right", "from": [13.0, 1.0, 0.0]}
```

### wall_left（沿Z轴，从-6到8）

**当前（错误）**：
```json
{"id": "window_left", "from": [-8.0, 1.0, -2.0]}
```

**修正后**：
```json
{"id": "window_left", "from": [4.0, 1.0, 0.0]}  // -2-(-6)=4
```

### wall_right（沿Z轴，从-6到8）

**当前（错误）**：
```json
{"id": "window_right", "from": [8.0, 1.0, 2.0]}
```

**修正后**：
```json
{"id": "window_right", "from": [8.0, 1.0, 0.0]}  // 2-(-6)=8
```

### 其他墙体

类似方法计算所有开口的正确坐标。
