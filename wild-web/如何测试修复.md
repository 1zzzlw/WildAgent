## 常见问题

### Q1: 为什么初始显示"0个网格"？

**A:** 这是正常的！应用启动时加载的是空场景。你需要手动加载 bieshu.wild 文件。

### Q2: 加载文件后还是看到旧的错误渲染？

**A:** 说明代码没有更新。可能原因：

1. **dev server没有运行** - 检查CMD/PowerShell窗口是否显示Vite服务器正在运行
2. **浏览器缓存** - 按 `Ctrl + Shift + R` 强制刷新
3. **TypeScript编译错误** - 查看CMD/PowerShell窗口是否有编译错误

### Q3: 控制台没有看到🧱和📦的调试日志？

**A:** 可能原因：

1. **没有开口的墙** - 只有带门窗的墙才会输出
2. **代码没有更新** - 强制刷新浏览器
3. **墙体长度 ≤ 20米** - 我的调试代码设置了阈值

尝试加载 bieshu.wild，它有很多门窗，一定会触发调试输出。

### Q4: dev server启动失败，提示端口被占用？

**A:** 其他进程占用了5173端口。解决方法：

1. 关闭其他可能占用端口的程序
2. 或者修改 `vite.config.ts`，换一个端口：
   ```typescript
   export default defineConfig({
     server: {
       port: 3000  // 改成其他端口
     }
   })
   ```

### Q5: TypeScript编译报错？

**A:** 我的修改只会产生警告（未使用的变量），不会有错误。如果有错误，可能是文件保存不完整。

检查这些文件是否存在且完整：
- `src/wild-core/src/primitive/resolver.ts`
- `src/wild-core/src/primitive/index.ts`
- `src/components/viewport/CanvasViewport.vue`
