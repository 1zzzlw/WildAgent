## 详细修复说明

### 1. 地板厚度方向修复

**问题**: 地板从顶面向下偏移，导致下方有空隙

**修复**: `floor.ts` 第26行和第67行
```typescript
// 修复前
position: [midX, from[1] - effectiveThick/2, midZ]

// 修复后
position: [midX, from[1] + effectiveThick/2, midZ]
```

**影响**: 所有地板（ground_floor, upper_floor, balcony等）

---

### 2. 开口Y坐标转换修复

**问题**: 蓝图中开口使用世界Y坐标，但传给墙体生成器时应该转换为相对墙底的偏移

**修复**: `resolver.ts` 第304行
```typescript
// 修复前
wall._cutouts.push({
  localX: opening.from[0],
  localY: opening.from[1],  // 直接用世界坐标
  ...
});

// 修复后
wall._cutouts.push({
  localX: opening.from[0],
  localY: opening.from[1] - wallFrom[1],  // 转换为相对偏移
  ...
});
```

**示例**:
```
墙体: from=[0, 3, 0], to=[6, 5.8, 0]  (墙底Y=3)
开口: from=[2, 4.0, 0]  (底部Y=4.0世界坐标)

修复前: localY = 4.0 → 孔在墙底向上4米（错误，超出墙高）
修复后: localY = 4.0 - 3.0 = 1.0 → 孔在墙底向上1米（正确）
```

**影响**: 所有开口，特别是二层及以上的门窗

---

### 3. 开口Y坐标重复加半高修复

**问题**: 开口几何体的Y坐标被加了两次半高度

**修复**: `opening.ts` 第18-27行
```typescript
// 修复前
const pos = worldPos || [params.from[0], params.from[1] + hh, params.from[2]];
return [{
  transform: {
    position: [pos[0], pos[1] + hh, pos[2]],  // 又加了一次hh
    ...
  }
}];

// 修复后
const centerY = worldPos ? worldPos[1] + hh : params.from[1] + hh;
const pos = worldPos || [params.from[0], params.from[1], params.from[2]];
return [{
  transform: {
    position: [pos[0], centerY, pos[2]],  // 只加一次hh
    ...
  }
}];
```

**影响**: 所有开口 - 导致门窗飘在墙上方

---

### 4. 墙体默认参数修复

**问题**: 构件库中墙体使用了错误的参数格式

**修复**: `BlockLibrary.vue` 第48行
```typescript
// 修复前
defaults: { 
  type: 'wall', 
  from: [0, 0, 0],      // Y=0
  to: [4, 0, 0],        // Y=0 (无高度)
  height: 3,            // 不存在的字段
  thickness: 0.24 
}

// 修复后
defaults: { 
  type: 'wall', 
  from: [0, 0, 0],      // 墙底
  to: [4, 3, 0],        // 墙顶 (高3米)
  thickness: 0.24,
  material: 'default'
}
```

**影响**: 构件库添加墙体功能

---

### 5. 地板默认参数修复

**问题**: 构件库中地板使用了不存在的 `region` 字段

**修复**: `BlockLibrary.vue` 第58行
```typescript
// 修复前
defaults: { 
  type: 'floor', 
  region: [[0, 0, 0], [4, 0, 0], [4, 0, 4], [0, 0, 4]],  // 不存在的字段
  thickness: 0.2 
}

// 修复后
defaults: { 
  type: 'floor', 
  from: [0, 0, 0],      // 矩形角点1
  to: [4, 0, 4],        // 矩形角点2
  thickness: 0.2,
  material: 'default'
}
```

**影响**: 构件库添加地板功能

## 技术说明

### 坐标系统总结

| 构件类型 | 坐标字段 | 含义 |
|---------|---------|------|
| 墙体 | from, to | 世界坐标 [X, Y, Z]，Y表示墙底和墙顶 |
| 地板 | from, to | 世界坐标，from是顶面Y，厚度向下延伸 |
| 开口 | from[0] | **沿墙距离**（非世界坐标） |
| 开口 | from[1] | **世界Y坐标**（非相对偏移） |
| 开口 | from[2] | 法向偏移（通常为0） |

### 为什么这些bug之前没被发现？

1. **地板厚度**:
   - 单层建筑的地板通常在Y=0附近，向上或向下偏移视觉差异不大
   - 只有多层建筑或细看时才明显

2. **开口Y坐标**:
   - 一层建筑墙底通常是Y=0，`opening.from[1] - 0 = opening.from[1]`
   - 一层建筑的开口位置碰巧正确
   - 只有二层及以上（墙底Y≠0）才暴露问题

3. **开口重复加半高**:
   - 如果bug #2存在，bug #3的错误被部分抵消
   - 只有修复bug #2后，bug #3才完全暴露

4. **构件库参数**:
   - 早期可能没有构件库功能
   - 或只测试了柱子和屋顶，没测试墙体和地板
