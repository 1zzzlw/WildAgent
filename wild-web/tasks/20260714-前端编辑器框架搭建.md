# 2026年7月14日 - 前端编辑器框架搭建

## 任务概述

从零搭建 WILD 建筑编辑器前端完整框架，基于 Vue 3 + Vite + TypeScript + Three.js + Pinia，实现状态管理、组件系统、ScenePatch 协议和基础编辑功能。

2026年7月14日

- **框架**: Vue 3 (Composition API) + TypeScript

- **3D 渲染**: Three.js

### 2. 类型系统定义

#### Blueprint 核心类型

**文件**: `wild-web/src/types/blueprint.ts`

定义了 WILD 语言的核心数据结构：

```typescript
interface Blueprint {
  meta: BlueprintMeta           // 元数据
  geometry: GeometrySection     // 几何定义
  materials?: MaterialDef       // 材质库
  behaviors?: BehaviorsSection  // 行为定义
  editor?: EditorMetadata       // 编辑器元数据
}
```

支持的构件类型：
- `wall`: 墙体
- `floor`: 地板
- `column`: 柱子
- `beam`: 梁
- `roof`: 屋顶
- `opening`: 门窗
- `stair`: 楼梯
- `furniture`: 家具
- `dense_brick`: 密集砖块
- `body`: 化身

#### ScenePatch 协议

**文件**: `wild-web/src/types/scenePatch.ts`

定义了统一的场景修改协议：

```typescript
interface ScenePatch {
  type: 'scene_patch'
  patch_id: string
  base_revision: number
  source: 'user' | 'agent' | 'system'
  mode: 'apply' | 'proposal'
  requires_confirmation: boolean
  operations: SceneOperation[]
  summary?: string
}
```

支持的操作类型：
- `add_element`: 添加构件
- `update_element`: 修改构件参数
- `remove_element`: 删除构件
- `upsert_material`: 添加/更新材质
- `add_template / update_template / remove_template`: 模板操作
- `add_instance`: 添加实例
- `add_placement`: 批量放置
- `set_behaviors`: 设置行为
- `set_editor_meta`: 设置编辑器元数据

#### 场景文档类型

**文件**: `wild-web/src/types/scene.ts`

定义场景文档、重建实体、网格数据等：

```typescript
interface SceneDocument {
  id: string
  name: string
  revision: number          // 版本号
  blueprint: Blueprint
  dirty: boolean           // 是否有未保存修改
  savedAt?: number
}

interface ReconstructedEntity {
  meshes: MeshData[]
  materialParams?: Record<string, unknown>
  boundingBox?: BoundingBox
  physics?: unknown
  scripts?: unknown
  animations?: unknown
}
```

#### Agent 通信协议

**文件**: `wild-web/src/types/agent.ts`

定义 WebSocket 通信消息类型：

- `UserMessageRequest`: 前端发送的用户消息
- `AgentStepResponse`: Agent 执行步骤
- `PatchProposalResponse`: Patch 提案
- `AgentReplyResponse`: Agent 文本回复
- `ErrorResponse`: 错误响应
- `ChatMessage`: 聊天消息
- `AgentSession`: 会话状态

### 3. 状态管理系统（Pinia Stores）

#### sceneStore - 场景状态

**文件**: `wild-web/src/stores/sceneStore.ts`

管理场景文档、Blueprint、重建结果：

- `document`: 当前场景文档
- `reconstructed`: 重建后的几何数据
- `validationIssues`: 校验问题列表
- `isReconstructing`: 是否正在重建

核心方法：
- `createEmptyDocument()`: 创建空场景
- `loadBlueprint()`: 加载 Blueprint
- `applyPatch()`: 应用 ScenePatch（包含 revision 检查、校验）
- `reconstruct()`: 重建场景（调用 wild-core）
- `validate()`: 校验 Blueprint
- `exportWild()`: 导出 .wild 文件
- `markSaved()`: 标记为已保存

#### selectionStore - 选择状态

**文件**: `wild-web/src/stores/selectionStore.ts`

管理构件选择和悬停：

- `selectedIds`: 选中的构件 ID 列表
- `hoveredId`: 悬停的构件 ID

核心方法：
- `select()`: 选中构件（支持多选）
- `deselect()`: 取消选中
- `clearSelection()`: 清空选择
- `toggleSelection()`: 切换选中状态
- `setHovered()`: 设置悬停
- `isSelected()` / `isHovered()`: 检查状态

#### historyStore - 撤销重做

**文件**: `wild-web/src/stores/historyStore.ts`

管理操作历史：

```typescript
interface HistoryEntry {
  label: string
  before: Blueprint
  after: Blueprint
  patch: ScenePatch
  timestamp: number
}
```

核心方法：
- `push()`: 添加历史记录（最多保存 50 条）
- `undo()`: 撤销
- `redo()`: 重做
- `clear()`: 清空历史
- `getCurrentState()`: 获取当前状态

#### agentStore - AI 对话

**文件**: `wild-web/src/stores/agentStore.ts`

管理 Agent 会话和消息：

- `session`: 会话信息（session_id, messages, connected）
- `pendingPatch`: 待确认的 patch
- `isProcessing`: Agent 是否处理中
- `currentStep`: 当前执行步骤

核心方法：
- `addUserMessage()` / `addAgentMessage()` / `addSystemMessage()`
- `setProcessing()`: 设置处理状态
- `setPendingPatch()`: 设置待确认 patch
- `confirmPatch()` / `rejectPatch()`: 确认/拒绝 patch
- `setConnected()`: 设置连接状态
- `resetSession()`: 重置会话

#### uiStore - UI 布局

**文件**: `wild-web/src/stores/uiStore.ts`

管理面板显示和尺寸：

- 左侧面板：场景树 / 构件库
- 右侧面板：属性 / 校验
- 底部面板：AI 对话
- 面板宽度/高度配置

### 4. 核心逻辑实现

#### ScenePatch 应用

**文件**: `wild-web/src/wild/scenePatch.ts`

实现 Patch 应用逻辑：

```typescript
function applyPatchToBlueprint(
  blueprint: Blueprint, 
  patch: ScenePatch
```

支持所有操作类型的应用，包括：
- 添加/修改/删除元素
- 材质管理
- 模板操作
- 实例化和批量放置

辅助函数：
- `createPatch()`: 创建 ScenePatch 对象

#### Blueprint 校验

**文件**: `wild-web/src/wild/sceneValidator.ts`

实现 Blueprint 校验：

```typescript
function validateBlueprint(blueprint: Blueprint): ValidationIssue[]
```

校验规则：
- 检查 meta.version 和 meta.type 必填
- 检查构件 id 唯一性
- 检查构件 type 必填
- 校验材质引用存在性
- 校验 opening.parentWall 存在性

返回 `ValidationIssue[]`（level: 'error' | 'warning'）

#### 场景摘要生成

**文件**: `wild-web/src/wild/sceneSummary.ts`

生成场景摘要用于 Agent：

```typescript
function generateSceneSummary(blueprint: Blueprint): SceneSummary
```

包含：
- `elements_count`: 元素数量
- `types`: 构件类型列表
- `bbox`: 边界盒
- `materials`: 材质列表

#### ID 生成工具

**文件**: `wild-web/src/wild/idFactory.ts`

生成唯一构件 ID：

```typescript
function generateElementId(type: string, existingIds: Set<string>): string
function generateUniqueId(prefix: string): string
```

#### 默认值定义

**文件**: `wild-web/src/wild/blueprintDefaults.ts`

提供各类构件的默认参数：

```typescript
function createEmptyBlueprint(name?: string): Blueprint
function getElementDefaults(type: string): Record<string, unknown>
```

支持的默认值：
- wall: from, to, height, thickness
- column: base, height, bottomRadius, topRadius
- floor: region, thickness
- beam: from, to, width, height
- roof: roofType, span, depth, height
- opening: parentWall, from, width, height
- furniture: subtype, position, dimensions

基于 Pinia 的集中式状态管理：
- 清晰的职责划分（scene / selection / history / agent / ui）

- 易于调试和测试

### 4. 组件化架构

- 布局与功能分离

- 易于扩展

- 深色主题

- 平滑的过渡动画
