# 🎯 最终修复 - 边界盒问题根源

## 问题诊断

你的日志显示边界盒异常大：
```
边界盒尺寸: X=22.3m, Y=9.4m, Z=30.6m
```

虽然墙体上的洞（cutouts）坐标已经修复，但**门窗mesh本身的位置**还在使用错误的坐标！

## 根本原因

在 `resolveOpenings` 函数中，有**两个地方**使用开口坐标：

### 1. ✅ Cutout坐标（已修复）
用于在墙体上开洞 - **已在之前修复**

### 2. ❌ 门窗mesh位置（刚刚修复）
用于放置门窗mesh - **这是导致边界盒异常的真正原因！**

## Bug位置

`resolver.ts` 第360-364行：

```typescript
// ❌ 错误：直接使用世界坐标opening.from[0]
const worldX = wallFrom[0] + opening.from[0] * dirX + opening.from[2] * nx;
const worldZ = wallFrom[2] + opening.from[0] * dirZ + opening.from[2] * nz;
```

例如 `window_front_left`:
- `opening.from[0] = -5.0` (世界X坐标)
- 但代码把它当作沿墙距离使用
- 导致：`worldX = -8 + (-5) * 1 = -13` ❌
- 正确：`worldX = -8 + 3 * 1 = -5` ✓

## 修复方案

使用计算好的 `localX` 和 `localY`：

```typescript
// ✓ 正确：使用localX（沿墙距离）
const worldX = wallFrom[0] + localX * dirX + opening.from[2] * nx;
const worldZ = wallFrom[2] + localX * dirZ + opening.from[2] * nz;
opening._worldPos = [worldX, wallFrom[1] + localY, worldZ];
```
