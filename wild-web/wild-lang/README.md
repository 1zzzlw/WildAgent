原语语言 (WILD) v1.0
原语是原荒 (Wild) 协议的数字资产描述语言，用于定义建筑和化身的外观、材质、物理和行为。它是一门声明式领域特定语言 (DSL)，不包含任何可执行逻辑。

核心原则
万物皆信息：所有视觉属性都是精确的数值参数，不存在纹理贴图。

创作者表达权完整：原语包含完整的视觉信息，所有合规客户端渲染结果完全一致。

文件体积极小：通过参数化构件和模板实例化，将 MB 级模型压缩到 KB 级。

独立于实现：本规范只定义语言语法和语义，不约束任何特定引擎的实现方式。任何引擎只要通过合规性测试即可接入原荒主网。

仓库结构
SPEC.md — 语法规范、顶层结构和字段定义

PRIMITIVES.md — 参数化构件库

MATERIALS.md — 材质与效果层

BEHAVIORS.md — 物理、动画和交互指令

VERSIONING.md — 版本兼容规则

schema.json — JSON Schema 校验定义

types.ts — TypeScript 类型参考

快速开始
示例：一根石柱
json
{
  "meta": { "version": "1.0", "type": "building", "name": "Stone Column" },
  "geometry": {
    "elements": [
      {
        "type": "column",
        "id": "col",
        "base": [0, 0, 0],
        "height": 4,
        "bottomRadius": 0.2,
        "topRadius": 0.15,
        "style": "modern",
        "material": "stone"
      }
    ]
  },
  "materials": {
    "stone": {
      "baseColor": [0.6, 0.6, 0.55],
      "roughness": 0.8,
      "metallic": 0.0,
      "albedo": 1.0,
      "lightingCondition": "D65_noon"
    }
  }
}
任何通过合规性测试的原语引擎都必须对这份蓝图输出完全相同的网格和材质参数。

语言版本
当前版本："1.0"。蓝图通过 meta.version 字段声明其使用的语言版本。原语引擎读取该字段以选择正确的解析规则。

版本升级遵循"永不删除"原则：更高的语言版本只增加新构件和字段，不删除或修改任何已有定义。v1.0 中合法的蓝图在 v2.0 中继续有效且语义不变。

合规性
合规性由原荒协议的合规性测试套件保证。任何声称实现本语言规范的引擎，必须通过该套件中所有测试蓝图的验证，否则无法接入主网。
