## 修复原则

每次修复需记录：
1. **现象**：用户/渲染看到了什么错误
2. **根因**：哪段代码、什么逻辑导致
3. **影响范围**：哪些蓝图文件受影响
4. **修复方案**：改了什么、为什么这样改
5. **验证方式**：如何确认修复有效
