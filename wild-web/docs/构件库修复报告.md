# 构件库修复报告 (2026-07-21)

## 问题描述

用户反馈构件库中的墙体和地板无法正常渲染：
1. **墙体问题**：添加墙体后显示为地板（水平平面）
2. **地板问题**：添加地板后导致其他构件消失

## 根本原因

构件库（`BlockLibrary.vue`）中的默认参数格式**不符合类型定义**。

### Bug 1: 墙体默认参数错误 ❌

**错误定义** (第48行):
```typescript
defaults: { 
  type: 'wall', 
  from: [0, 0, 0],      // Y=0 (墙底)
  to: [4, 0, 0],        // Y=0 (墙顶) ❌ 墙顶和墙底Y相同
  height: 3,            // ❌ WallParams没有height字段
  thickness: 0.24 
}
```

**问题**：
- `from` 和 `to` 的Y坐标都是0，墙体没有高度
- 添加了 `height: 3` 字段，但 `WallParams` 接口中不存在此字段
- 结果：渲染引擎只能生成一个水平平面（高度=0）

**正确定义**：
```typescript
defaults: { 
  type: 'wall', 
  from: [0, 0, 0],      // 墙底：Y=0
  to: [4, 3, 0],        // 墙顶：Y=3 ✓ 墙高3米
  thickness: 0.24,
  material: 'default'
}
```

**类型定义参考** (`wild-core/types.ts`):
```typescript
export interface WallParams {
  type: 'wall';
  id: string;
  from: Vec3;    // [X, Y, Z] - 墙底坐标
  to: Vec3;      // [X, Y, Z] - 墙顶坐标 (Y应该 > from[1])
  thickness: number;
  material?: string;
}
```

### Bug 2: 地板默认参数错误 ❌

**错误定义** (第58行):
```typescript
defaults: { 
  type: 'floor', 
  region: [[0, 0, 0], [4, 0, 0], [4, 0, 4], [0, 0, 4]],  // ❌ 不存在region字段
  thickness: 0.2 
}
```

**问题**：
- 使用了 `region` 字段（多边形点数组），但 `FloorParams` 接口中不存在此字段
- 渲染引擎需要 `from` 和 `to` 两个对角点来定义矩形地板
- 结果：渲染引擎无法解析，可能导致错误或忽略该构件

**正确定义**：
```typescript
defaults: { 
  type: 'floor', 
  from: [0, 0, 0],      // 矩形区域角点1
  to: [4, 0, 4],        // 矩形区域角点2 (对角)
  thickness: 0.2,
  material: 'default'
}
```

**类型定义参考** (`wild-core/types.ts`):
```typescript
export interface FloorParams {
  type: 'floor';
  id: string;
  from: Vec3;    // 矩形区域角点1
  to: Vec3;      // 矩形区域角点2 (对角)
  thickness: number;
  material?: string;
  shape?: 'rect' | 'circle';  // 可选：圆形地板
  radius?: number;            // 圆形地板半径
}
```

## 技术说明

### 为什么不用region定义地板？

**历史原因**：
- 可能早期版本支持过 `region`（多边形顶点数组）
- 但当前引擎只支持矩形地板（`from/to`）和圆形地板（`radius`）

**设计考虑**：
- 矩形地板最常用，用两个对角点定义最简单
- 圆形地板用中心点+半径定义
- 复杂多边形地板可以用多个矩形拼接

### 为什么墙体不用height字段？

**设计决策**：
- 墙体使用 `from/to` 统一表示起点和终点
- `from[1]` = 墙底Y坐标，`to[1]` = 墙顶Y坐标
- 高度 = `to[1] - from[1]`（自动计算）
- 这样可以支持不同高度的墙（如二层墙从Y=3开始）

**优势**：
- 与屋顶、梁等构件的坐标系统一致
- 支持复杂建筑（多层、错层）
- 避免额外的高度字段
