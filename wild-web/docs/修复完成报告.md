## 修复方案

### 1. 别墅（bieshu.wild）

**问题**：侧墙的Z坐标与前后墙不一致
```json
// 前墙：-6.0 到 -6.0
// 后墙：8.0 到 8.0
// 侧墙：-5.7 到 7.7  ❌ 差了0.3米（墙厚）
```

**修复**：统一所有墙体的Z坐标范围
```json
"wall_left": {
  "from": [-8.0, 0.0, -6.0],  // 原来是 -5.7
  "to": [-8.0, 3.0, 8.0]       // 原来是 7.7
}
```

影响墙体：
- `wall_left`（1层）
- `wall_right`（1层）
- `wall_interior_ground`（1层）
- `wall_upper_left`（2层）
- `wall_upper_right`（2层）
- `wall_upper_interior`（2层）

### 2. 天坛（tiantan.wild）

**问题**：4个门在圆周墙上的位置不均匀
```
实际：15°, 105°, 195°, 293°
期望：0°, 90°, 180°, 270°
```

**修复**：重新计算弧长坐标
- 半径：5.8米
- 周长：2πr ≈ 36.44米
- 90°间隔：36.44/4 = 9.11米

```json
"door_N": {"from": [0, 0, 0], ...},      // 0° (原来1.52)
"door_E": {"from": [9.11, 0, 0], ...},   // 90° (原来10.63)
"door_S": {"from": [18.22, 0, 0], ...},  // 180° (原来19.74)
"door_W": {"from": [27.33, 0, 0], ...}   // 270° (原来28.85)
```

## 技术总结

### 为什么DeepSeek的修复没有解决问题？

1. **诊断正确**：渲染引擎确实有4个bug
2. **修复正确**：代码修改都是对的
3. **但不够**：引擎bug修复后，蓝图文件的几何错误依然存在

类比：
- 引擎bug = 计算器的算法错误
- 蓝图错误 = 输入了错误的数字
- 修复引擎 = 修好了计算器
- 但输入仍然错误 = 结果还是错的

### 根本原因

渲染引擎只负责：
- 根据蓝图**计算**几何位置
- 自动**推导**空间关系
- **修复微小偏差**（< 0.01米）

但不会：
- **纠正**蓝图的设计错误
- **重新设计**建筑布局
- **猜测**用户意图

当墙角gap达到0.3米时，引擎认为这是"有意设计"，不会自动修复。
