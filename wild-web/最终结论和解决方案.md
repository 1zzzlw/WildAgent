# 最终结论和解决方案

## 问题根源

**bieshu.wild蓝图文件使用了错误的开口坐标格式**

### 正确格式（cabin_v1使用的）

```json
{
  "type": "opening",
  "parentWall": "front_wall",  // 墙从[0,0.5,0]到[6,3.5,0]
  "from": [2.4, 0.5, 0],       // [沿墙距离, Y坐标, 法向偏移]
}
```

### 错误格式（bieshu使用的）

```json
{
  "type": "opening",
  "parentWall": "wall_front",  // 墙从[-8,0,-6]到[8,3,-6]
  "from": [-5.0, 1.0, -6.0],   // ❌ [世界X, 世界Y, 世界Z]
}
```

## 为什么cabin_v1正常？

**cabin_v1从一开始就用了正确的格式**，所以原始引擎可以正确渲染。

## 为什么bieshu错误？

**bieshu使用了错误的格式**，原始引擎无法正确处理。

## 为什么我的修复破坏了cabin_v1？

我尝试修改引擎来"自动识别"坐标格式，但这是不可能的，因为：
- 无法区分"沿墙2.4米"和"世界X坐标2.4"
- 导致把正确的cabin_v1也当成错误格式处理了

## 正确的解决方案

**修复bieshu.wild蓝图文件**，把所有开口的from改成正确格式。

## 如何修复bieshu.wild

### 计算公式

对于直线墙：
```
沿墙距离 = opening世界坐标 - wall起点坐标（在墙方向上的分量）
```
