## 为什么cabin_v1正常而bieshu有问题？

**cabin_v1的蓝图文件使用了正确的格式（沿墙距离）**

例如cabin_v1中：
```json
{
  "type": "opening",
  "id": "front_door",
  "parentWall": "front_wall",
  "from": [2.4, 0.0, 0.0]  ← 这是沿墙2.4米，不是世界坐标
}
```

front_wall从[0,0.5,0]到[6,3.5,0]，开口在沿墙2.4米处是合理的。

**bieshu的蓝图文件错误地使用了世界坐标**，这可能是手动编辑时的错误，或者是不同版本蓝图编辑器生成的。
