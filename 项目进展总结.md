# WildAgent 项目进展总结

> 最后更新：2026-07-23

## 1. 项目概述

**WildAgent** 是一个 AI 辅助的参数化 3D 建筑编辑器。用户用自然语言描述建筑，系统自动生成 `.wild` 蓝图文件并实时渲染 3D 效果。

### 核心理念

| 原则 | 说明 |
|---|---|
| `.wild` 是唯一场景源文件 | 所有建筑结构、材质、行为最终都落到 `.wild` JSON |
| 渲染引擎只复用不重做 | `wild-core` 是现有核心引擎，新系统只接入它 |
| AI 不直接生成三角形 | AI 只生成参数、构件、材质、patch |
| 用户保留最终决策权 | 批量修改、删除、覆盖场景必须用户确认 |
| 手动编辑不走 Agent | 拖拽、属性面板、场景树操作直接修改本地状态 |

不同于主流 AI 3D 生成工具（Meshy/Tripo）直接生成三角网格，WildAgent 采用：
- **AI 只填参数表**，不生成几何
- **数学引擎负责计算**，保证精确和可复现
- **产物是活参数文件**（几 KB），不是死网格文件（几十 MB）
- **完全可编辑**，每个参数独立可调

### 三大模块

```
┌─────────────────┐
│   wild-core     │  已有：渲染/重建引擎（不重做）
│  (引擎核心)      │  输入 .wild → 输出几何
└─────────────────┘

┌─────────────────┐
│   wild-web      │  新建：前端编辑器
│  (前端编辑器)    │  UI + 状态管理 + Three.js 渲染
└─────────────────┘

┌─────────────────┐
│   wild-server   │  新建：后端 Agent 服务
│  (AI 服务)       │  理解意图 → 生成 .wild / Patch
└─────────────────┘
```

## 4. 核心架构设计

### 4.1 Blueprint（蓝图）

`.wild` 文件的 JSON 格式，是场景的唯一源文件。包含：
- `meta`：版本/类型/名称
- `geometry`：构件数组（elements）
- `materials`：材质定义
- `behaviors`：行为脚本

### 4.2 ScenePatch（场景补丁）

所有修改操作的统一格式，用户操作和 Agent 建议都转为 ScenePatch：

```typescript
interface ScenePatch {
  type: 'scene_patch'
  patch_id: string
  base_revision: number        // 基于哪个版本，不匹配则拒绝
  source: 'user' | 'agent' | 'system'
  mode: 'apply' | 'proposal'
  requires_confirmation: boolean
  operations: SceneOperation[]  // add/update/remove/upsert
  summary?: string
}
```

- 每次成功应用后 `revision++`
- Patch 的 `base_revision` 必须等于当前 revision，防止冲突

### 4.3 数据流

```
用户操作 / Agent 建议 → ScenePatch → sceneStore.applyPatch()
  → Blueprint 更新 → 校验
  → wild-core.reconstructEntity()
  → MeshData + MaterialParams → Three.js 渲染
```

## 5. 前端关键模块

### 状态管理（5 个 Pinia Store）

| Store | 职责 |
|---|---|
| `sceneStore` | 场景文档 + Blueprint + 重建触发 |
| `selectionStore` | 选中 / hover 状态 |
| `historyStore` | 撤销/重做（50 步历史） |
| `agentStore` | AI 对话消息 + WebSocket 连接状态 |
| `uiStore` | UI 面板布局状态 |

### 渲染管线

```
wildCoreAdapter (parse + reconstruct)
  → meshDataToGeometry (MeshData → BufferGeometry)
  → materialAdapter (MaterialParams → Three.js 材质 + 缓存)
  → renderEntity (ReconstructedEntity → Three.js Group)
  → CanvasViewport (灯光 + 阴影 + 轨道控制 + 响应式)
```

### wild-core 几何构建器（10 种）

wall / floor / column / beam / roof / stair / opening / furniture / body / denseBrick

### Agent 通信

- `agentBridge.ts`：WebSocket 桥接（心跳 + 自动重连 + 指数退避 + 页面可见性检测）
- `AIChatPanel.vue`：AI 对话 UI（消息列表 + 输入框 + 思考动画）

### 架构备忘：wild-core 放置位置

wild-core（22 个 TS 文件）当前放在 `wild-web/src/wild-core/`。**现阶段合理，Phase 3 以后计划抽为独立 package（`@wild/core`）**。原因：
- 引擎不依赖任何前端框架，是独立的核心资产
- 目前改动频繁，放前端内调试方便；稳定后抽走更划算
- 抽取后可以有独立版本号和单元测试，wild-server 将来也能引用做服务端校验

详见 [架构设计方案.md §17.5](架构设计方案.md)。
